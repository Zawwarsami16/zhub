# Phase 4.0 — Brain adapters + multi-brain publisher

**Status:** spec
**Date:** 2026-05-09
**Decided in conversation:** all four brains in scope, streaming-first across all, code lives in `zhub/brains/` module, auto-detect with explicit override.

---

## Goal

Prove the **swappable-brain** claim concretely. Today every zhub publisher example wires one specific brain (Anthropic API in `claude_publisher.py`, MCP subprocess in `mcp_bridge.py`, file-bridge in `session_bridge_publisher.py`). Operators who want to swap from cloud Claude to free Ollama have to write their own publisher from scratch.

After this phase: a single example publisher reads CLI/env, picks a brain, and publishes. The brain adapters are reusable building blocks anyone can compose into their own publisher script.

## Non-goals

- Tool-call resolution inside the adapter (hub already does that on the chat-completions path; adapters just stream raw assistant text).
- Multi-brain at the same publisher (one publisher process = one active brain — for failover/load-balance, run multiple publishers behind one entity, which is a separate Phase 4.x topic).
- A frontend / UI for picking brains. CLI flag + env var is the surface.

## Architecture

```
zhub/brains/
  __init__.py           — registry, detect(), list_available()
  base.py               — BrainAdapter ABC + small helpers
  ollama.py             — OllamaAdapter
  groq.py               — GroqAdapter
  openai.py             — OpenAIAdapter
  cerebras.py           — CerebrasAdapter
examples/
  multi_brain_publisher.py
```

`zhub/brains/__init__.py` exposes:

```python
from .base import BrainAdapter, ChatChunk
from .ollama import OllamaAdapter
from .groq import GroqAdapter
from .openai import OpenAIAdapter
from .cerebras import CerebrasAdapter

REGISTRY: list[type[BrainAdapter]] = [
    OllamaAdapter, GroqAdapter, OpenAIAdapter, CerebrasAdapter,
]

def detect() -> Optional[BrainAdapter]:
    """Return the first available adapter in priority order, or None."""
    for cls in REGISTRY:
        adapter = cls.try_init()
        if adapter is not None:
            return adapter
    return None

def list_available() -> list[BrainAdapter]:
    return [a for cls in REGISTRY if (a := cls.try_init()) is not None]
```

Priority order: Ollama → Groq → OpenAI → Cerebras (cheapest local first).

## BrainAdapter interface

```python
@dataclass
class ChatChunk:
    delta: str = ""              # incremental text
    done: bool = False
    finish_reason: Optional[str] = None
    raw: Optional[dict] = None    # the underlying API's chunk, for debug

class BrainAdapter(abc.ABC):
    name: str                     # "ollama", "groq", ...
    label: str                    # human label

    @classmethod
    @abc.abstractmethod
    def try_init(cls) -> Optional["BrainAdapter"]:
        """Return an instance if credentials/endpoint are configured AND
        the service responds to a quick probe. Return None otherwise.
        Probes time out fast (<1s) so detection stays snappy."""

    @abc.abstractmethod
    async def stream(
        self,
        messages: list[dict],
        *,
        system: Optional[str] = None,
        temperature: float = 0.7,
        max_tokens: int = 2048,
        tools: Optional[list[dict]] = None,
    ) -> AsyncIterator[ChatChunk]:
        """Stream the model's reply as ChatChunks. Final chunk has done=True."""
```

Adapters are lazily instantiated in `try_init()` so importing `zhub.brains` doesn't open network connections; only `detect()` / `list_available()` do.

## Per-adapter detection rules

| Adapter | Signal it's available | Probe |
|---|---|---|
| Ollama | env `OLLAMA_HOST` set OR localhost:11434 reachable | `GET <host>/api/version`, expect 200 |
| Groq | env `GROQ_API_KEY` set | `GET /openai/v1/models` with key, expect 200 |
| OpenAI | env `OPENAI_API_KEY` set | `GET /v1/models` with key, expect 200 |
| Cerebras | env `CEREBRAS_API_KEY` set | `GET /v1/models` with key, expect 200 |

Probe timeout: 1.0s. Failures are silent — `try_init` returns None.

## Streaming wire format per adapter

Each adapter normalizes its native streaming format into `ChatChunk`:

- **Ollama** native: line-delimited JSON `{message:{content:"..."}, done:false}` from `POST /api/chat?stream=true`. Adapter accumulates `message.content` deltas.
- **Groq / OpenAI / Cerebras**: SSE OpenAI shape `{choices:[{delta:{content:"..."}}]}`. Adapter parses `data: ` lines, extracts `delta.content`.

All adapters yield a final `ChatChunk(done=True, finish_reason=...)` when upstream signals end-of-stream.

## Optional dependencies

`pyproject.toml` gains `[brains]` extras group:

```toml
[project.optional-dependencies]
brains = [
    "httpx>=0.27",       # already a dev dep; promote to brains required
    "openai>=1.50",       # for OpenAIAdapter + Groq (groq SDK is OpenAI-shaped)
    "groq>=0.13",         # native groq SDK with streaming
    "cerebras-cloud-sdk>=1.18",
    # Ollama uses raw httpx — no SDK needed
]
```

Adapters check imports inside `try_init()` and return None if the SDK isn't installed (so missing optional deps degrade gracefully instead of erroring).

## Example publisher

`examples/multi_brain_publisher.py`:

```python
import argparse, asyncio, os
from zhub import publish
from zhub.brains import detect, list_available, REGISTRY

async def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--brain", default="auto",
                        help="ollama|groq|openai|cerebras|auto")
    parser.add_argument("--name", default=os.environ.get("ZHUB_NAME", "me"))
    parser.add_argument("--hub", default=os.environ.get("ZHUB_HUB",
                                                         "ws://127.0.0.1:8080"))
    parser.add_argument("--list", action="store_true",
                        help="list available brains and exit")
    args = parser.parse_args()

    if args.list:
        for a in list_available():
            print(f"{a.name:10} {a.label}")
        return

    if args.brain == "auto":
        brain = detect()
        if brain is None:
            raise SystemExit("no brains available — set OLLAMA_HOST or "
                             "GROQ_API_KEY / OPENAI_API_KEY / CEREBRAS_API_KEY")
    else:
        match = next((cls for cls in REGISTRY if cls.name == args.brain), None)
        if match is None:
            raise SystemExit(f"unknown brain: {args.brain}")
        brain = match.try_init()
        if brain is None:
            raise SystemExit(f"{args.brain} not available — check creds/host")

    print(f"using brain: {brain.label}")

    async def chat_handler(messages, options):
        # extract optional system message from messages, fold into system
        system_parts = [m["content"] for m in messages if m.get("role") == "system"]
        non_system = [m for m in messages if m.get("role") != "system"]
        system = "\n\n".join(system_parts) if system_parts else None
        async for chunk in brain.stream(
            non_system, system=system,
            temperature=options.get("temperature", 0.7),
            max_tokens=options.get("max_tokens", 2048),
            tools=options.get("tools"),
        ):
            if chunk.delta:
                yield chunk.delta

    pub = publish(
        name=args.name,
        description=f"served by {brain.label}",
        chat_handler=chat_handler,
        hub_url=args.hub,
        public=True,
        api_key=os.environ.get("ZHUB_API_KEY") or None,
    )
    while not pub.api_key:
        await asyncio.sleep(0.05)
    print(f"key={pub.api_key}")
    await asyncio.Event().wait()

if __name__ == "__main__":
    asyncio.run(main())
```

The chat_handler is an async generator yielding strings — zhub's existing `_handle_chat` already supports this (handles `inspect.isasyncgen(result)` path with streaming-aware emit).

## Entity-awareness

When an adapter starts, it can optionally fetch `<hub>/entity` once and prepend the markdown to its system prompt — so Ollama (which has no inherent zhub knowledge) becomes immediately fluent in zhub. We expose this via a helper:

```python
async def fetch_entity_for_system_prompt(hub_url: str) -> Optional[str]:
    """Fetch /entity from the hub, return as a system-prompt prefix.
    Silent on failure — entity injection is opt-in convenience."""
```

The example publisher uses this opt-in via `--inject-entity` flag (default: off, so behavior matches existing publishers unless explicitly turned on).

## Out-of-scope (for next phases)

- Multi-brain hot-swap inside one publisher (single-process load-balance) — Phase 4.0b if needed.

- Failover at the hub layer (entity-level: multiple publishers serve the same entity, hub routes to first available) — Phase 4.x.

## Risks

- **SDK version churn**: Groq/OpenAI/Cerebras SDKs change. Pin minimum versions; tests use mocked HTTP, not the real SDKs, so SDK upgrades don't break CI.
- **Ollama probe is localhost-default**: on machines where Ollama's port is firewalled, detection fails. Documented in adapter's docstring.
- **Streaming → non-streaming fallback**: if upstream returns a non-streaming response (rare, but Ollama can do this), adapter wraps the full text as a single chunk.
